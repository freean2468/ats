# Type: Commit의 성격을 나타내며 선택할 수 있는 값은 아래와 같다.
#   Feat: 새로운 기능 추가
#   Fix: 버그 수정
#   Test: 테스트 코드 작성 혹은 수정
#   Migration: DB Migration File 추가(해당 Commit을 Pull하면 DB Migrate를 진행해야 함)
#   Config: 환경설정 수정
#   Refactor: 코드 리팩토링
#   File: .py 외의 File 관련(Dockerfile, requirements.txt 등)
#   Style: 코드 의미에 영향을 주지 않는 수정(Import Optimize, newline 등)
#   Annotation: 주석 작성 혹은 수정
#   Infra : Infra 수정
#   Workflow: GitHub Actions workflow 수정
#

#   Commit의 경우, commit: <commit id>
#   Issue의 경우, Issue: <issue id>
